Apps. Solutions. One vocabulary.

An app is software we ship. A solution is what a customer achieves with it. Don't mix them up.

Terminology · the apps-versus-solutions distinction
A
App

Software we ship.

Concrete, installable, one click from the Nextcloud app store. Has a name, a version, a maintainer, a GitHub repo.

e.g. OpenCatalogi · OpenRegister · OpenConnector · DocuDesk

S
Solution

What a customer achieves.

A concrete outcome, a problem solved, a regulation met, a workflow live. Built on apps, but not an app itself.

e.g. WOO compliance · Organisation registry · Case handling

One app can contribute to several solutions. One solution usually combines several apps. We speak from both sides: apps for visitors who know what they're looking for, solutions for visitors with a problem.

Never write "our WOO app", it doesn't exist. Write "our WOO solution, built on OpenCatalogi, OpenConnector, and DocuDesk."

Demos · partners run them, Conduction doesn't
Conduction builds apps. Partners install, demo, and support them. Anyone who wants to see a demo talks to a partner. We make sure the software works, they make sure the customer can use it.

Never write "Book a demo" or "Try the demo". Write "Request a demo from a partner" (EN) or "Vraag een demo aan bij een partner" (NL). Always link to the partner page.

Sales pitch · open source, free, GDPR. Not NLDS.
NLDS conformance is a fact, not a selling point. Our real proposition is open source, free to install, and GDPR-compliant by default. That's what the SMB decision-maker chooses, not a design-system stamp.

In heroes, badges, and ledes write "open source, free, GDPR-compliant". We mention NLDS where it actually matters: a filter, a spec row, the NLDesign theme app. Not as pride in a banner.
