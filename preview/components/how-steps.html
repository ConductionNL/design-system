How steps · numbered process row, surfaced from /support and /install
1

Read the docs first.

Every app at docs.conduction.nl ships with installation, configuration, schema, and recipe guides. The /docs section covers what we'd answer first in support tickets.

2

Open a GitHub issue.

For bugs, feature requests, and questions that aren't in the docs. We triage daily on the ConductionNL org. Public issues stay public, security issues go through coordinated disclosure.

3

Talk to a partner.

For implementations, hosting, and regulated rollouts. Partners take it from "I want this" to "this is in production". The /partners directory lists every certified team.

React / MDX usage

Mount in MDX as <HowSteps/>

Numbered process-step row used wherever a flow goes "three things, in this order": /support help-routing, /install onboarding, the partner-programme tiers explainer. Children auto-number; the hex-num tints alternately by tier (cobalt, orange, cobalt-700).

Import & example

import {HowSteps, HowStep} from '@conduction/docusaurus-preset/components';

<HowSteps>
  <HowStep title="Read the docs first.">
    Every app at docs.conduction.nl ships with installation, configuration,
    schema, and recipe guides.
  </HowStep>
  <HowStep title="Open a GitHub issue.">
    For bugs, feature requests, and questions that aren't in the docs.
  </HowStep>
  <HowStep title="Talk to a partner.">
    For implementations, hosting, and regulated rollouts.
  </HowStep>
</HowSteps>

Props

  • columns 2 | 3 | 4 (default 3)
  • HowStep.number auto-derived if omitted
  • HowStep.tier 1 | 2 | 3, override the auto-tint
  • HowStep.title h3 title
  • HowStep.children body (block-safe div, MDX-friendly)
