Why we built it this way
Pair · the integration headline

The orange arrow is the brand element

Every Conduction integration page opened with the same pattern, hand-rolled in each docs site: two hexes, a bridge, a centred orange arrow. cn-pair encodes that pattern so the brand rules can't drift.

  • One orange per scene, enforced. The arrow is always KNVB orange. The component owns the page's orange budget; place a second <cn-pair> or a <cn-hex color="orange"> on the same screen and the brand rule breaks. The component cannot stop you, but the static-kit specimen page documents the limit.
  • Attributes for strings, slots for SVG. Labels and captions are strings; passing them as attributes keeps the markup tight. Icons need DOM (gradients, multi-path, etc.), so they go through slots.
  • Sub-brand colours auto-correct ink. left-color="commonground" flips ink to cobalt-900 (WCAG AA on yellow); every other supported fill keeps white ink.
  • Stacks below 700px. Grid collapses to a single column on phones; the arrow rotates 90° so the visual flow keeps reading top-to-bottom.
  • No nested cn-hex. The hex inside cn-pair is rendered by the component, not a slotted cn-hex. This keeps the pair's two hexes in lockstep with the panel — same size, same vertical alignment, no surprise from a slotted element that decides to grow.
