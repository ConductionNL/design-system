# sites/connext

The Docusaurus 3 site for [connext.conduction.nl](https://connext.conduction.nl). Built on `@conduction/docusaurus-preset` for brand defaults; site-specific config lives in [docusaurus.config.js](docusaurus.config.js).

## Local development

Two ways to run the dev server: **Docker** (recommended; matches CI + production) or **bare Node** (faster iteration if your host already has Node 18+).

### Option A, Docker (recommended)

From the design-system repo root:

```bash
docker compose up                # build + start the connext container
                                  # → http://localhost:3000

docker compose up -d              # detached mode
docker compose logs -f connext    # tail the dev server
docker compose down               # stop + clean up
```

The container bind-mounts the repo source, so saves to `sites/connext/`, `diagrams/`, or `docusaurus-preset/` trigger Docusaurus hot-reload automatically. Node modules live in named Docker volumes; the host stays clean.

To wire the container to your local Nextcloud (for the future OpenCatalogi content plugin), uncomment the `networks:` block in [docker-compose.yml](../../docker-compose.yml) and set `OPENCATALOGI_URL=http://nextcloud:80/index.php/apps/openregister/api` in a `.env` file at the repo root.

### Option B, bare Node

From the design-system repo root:

```bash
# Install once (npm workspaces)
npm install

# Start the dev server
npm run dev:connext
# → http://localhost:3000/
```

Or from this folder:

```bash
npm start
```

## Building

```bash
npm run build:connext   # from repo root
npm run build           # from this folder
```

Output lands in `sites/connext/build/`. That folder is what gets deployed to GitHub Pages.

## Local Nextcloud integration

Once `@conduction/docusaurus-plugin-opencatalogi` is built, content can be pulled from the local Nextcloud OpenCatalogi register at build time:

```bash
# Start the local Nextcloud (Conduction's docker-compose dev environment)
cd ~/nextcloud-docker-dev && docker compose up -d

# Build the site against localhost:8080
OPENCATALOGI_URL=http://localhost:8080/index.php/apps/openregister/api npm run build:connext
```

The plugin is wired into [docusaurus.config.js](docusaurus.config.js) but commented out until the package exists.

## Structure

```
sites/connext/
  blog/                — Blog posts (NL canonical)
  docs/                — Doc pages
  i18n/                — Translation overrides per locale (nl/en/de/fr)
  src/
    css/site.css       — Site-specific CSS, appended to brand.css
    pages/index.jsx    — Homepage with cn-platform demo
  static/              — Static assets (favicon, images, CNAME)
  docusaurus.config.js — Site config (uses createConfig from preset)
  sidebars.js          — Auto-generated docs sidebar
```

## Languages

Four locales pinned, NL default:

- `/` → Nederlands (canonical)
- `/en/` → English
- `/de/` → Deutsch
- `/fr/` → Français

Markdown overrides per locale live in `i18n/<locale>/docusaurus-plugin-content-docs/current/`. UI strings come from `i18n/<locale>/code.json` (generated by `npm run write-translations`).
