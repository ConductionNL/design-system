AtomZones · five-atoms reference for the Conduction app chassis

Five zone cards, one per atom of the canonical chassis: Topbar, Left navigation, Main column, Page header, Sidebar. The focused atom is outlined in KNVB orange. The rest fades to 25% so the reader sees where each atom sits in context, not just what it looks like in isolation. The full chassis with all atoms enabled lives on app-mock.

.topbar

Topbar

App · Desk · always

The Nextcloud chrome row. Sits across every page unconditionally because every Conduction app lives inside Nextcloud's workspace. The shelf icons are the cross-app navigation; per-app links never go here.

.nav

Left navigation

App · required · Desk: never

The per-app sidebar. Carries this app's own primary navigation, plus a footer pinned to the bottom for global access (Settings, Feedback). The active item is the only place cobalt-100 backgrounds a row.

.col

Main column

App · Desk · always

The work surface. In an App pattern, the column opens with a .pageHeader (title + actions), then KPI strip and panels. In a Desk pattern, .col nests inside .grid to become a full-bleed widget canvas with no page header.

.pageHeader

Page header

Index · Detail · Desk: never

The first row of .col on every Index and Detail template. Carries the page title (left) and exactly two action buttons (right): one ghost, one primary. The header tracks .col's width: full-spread when the sidebar is closed, constrained when it is open.

.detail

Sidebar

App · optional · Desk: never

The right-hand sidebar. Optional, dismissible, anchored to the active record or the active view. Carries an icon + title + description in a header, then a tabbed body (Search / Columns by default). Class stays .detail for code; we call it the Sidebar in copy.

React / MDX usage

Mount in MDX as <AtomZones/>

Educational visual reference for the canonical Conduction app chassis. Renders five zone cards, each focusing on one atom (Topbar, Left navigation, Main column, Page header, Sidebar). Used on architecture pages in the per-app docs sites (nextcloud-vue.conduction.nl, procest.conduction.nl, openregister.conduction.nl) to teach where each atom sits in the chassis.

Self-contained, takes no props. The full chassis with every atom enabled lives at app-mock; this component is the per-atom teaching variant.

Import & example

import {AtomZones} from '@conduction/docusaurus-preset/components';

# The five atoms
