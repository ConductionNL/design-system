<Section spacing="default" background="tinted">
  <SectionHead
    eyebrow="How it works"
    title="Three steps, in order."
    align="stack"
    lede="Most questions don't need to reach us. Start with the docs. If you're stuck, open an issue. If you need someone to do it for you, talk to a partner."
  />
  <HowSteps>
    <HowStep title="Read the docs first.">
      Every app at <strong>docs.conduction.nl</strong> ships with installation, configuration, schema, and recipe guides. The /docs section covers what we'd answer first in support tickets.
    </HowStep>
    <HowStep title="Open a GitHub issue.">
      For bugs, feature requests, and questions that aren't in the docs. We triage daily on the <strong>ConductionNL</strong> org. Public issues stay public, security issues go through coordinated disclosure.
    </HowStep>
    <HowStep title="Talk to a partner.">
      For implementations, hosting, and regulated rollouts. Partners take it from "I want this" to "this is in production". The <strong>/partners</strong> directory lists every certified team.
    </HowStep>
  </HowSteps>
</Section>
