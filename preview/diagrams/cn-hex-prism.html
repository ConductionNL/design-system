API · slots
SlotPurpose
(default)The prism's name. Required. Shows on the top face as the primary label.
iconIcon SVG or <img>. Optional. Sits above the label, inherits currentColor (the family-700 ink).
kickerSmall uppercase line under the label. Used for category + status, e.g. "DATA · STABLE".
pillsOptional pill stack for feature chips floating on the top face.
API · attributes
AttributeValuesDefaultPurpose
familycobalt · coral · lavender · gold · mint · graycoralPastel family palette. Cobalt is reserved for the workspace (Nextcloud); six total. Each family auto-tints all three faces from -100 / -300 / -500 plus -700 ink.
sizesm (96px) · md (144px) · lg (192px) · xl (240px), or any CSS lengthmdWidth of the prism. Height is auto-derived from the SVG aspect ratio.
statecomingabsentRenders a "Coming" pill in the upper-right corner. Used for unreleased apps.
Why we built it this way
Atom · system carrier

The prism is the system. Side-boxes are everything else.

A flat hex (<cn-hex>) is a node label. A prism (<cn-hex-prism>) is a system component you can install. The shape difference carries the hierarchy.

  • Six families, one geometry. The shape is the same; only the four CSS variables (--face-top, --face-left, --face-right, --ink) change per family. Add a new family and every existing prism that requests it retints automatically.
  • Cobalt is reserved. Only the workspace (Nextcloud, the platform) gets cobalt. Apps and integrations always pick another family. This is the visual hierarchy.
  • Top face is the working surface. Icon, name, kicker, pills all sit on the lit face, never on the side faces. The padding-bottom: 18% on the content layer enforces this.
  • Face edges at 18% opacity. Subtle facet lines, not bold strokes. The prism reads as a 3D form, not as a drawn shape.
  • Three slots beyond the label. Icon + kicker + pills cover every case we have seen on platform diagrams; if you need something more, the diagram is wrong, not the prism.
  • Coming-state is one attribute. Add state="coming" on unreleased apps. The badge sits over the upper-right corner of the top face. No new variant per state.
Uses No dependencies. Built on inline SVG, no animation library yet. Used by cn-platform (planned), cn-pipeline (planned)
