Why we built it this way
Pattern · prisms = apps, boxes = everything else

The shape carries the hierarchy

A hex-prism is something you can install. A side-box is something you can't. That visual distinction is non-negotiable; bending it muddies the diagram.

  • White panel, cobalt-200 stroke, 6px radius. Quiet styling so the eye lands on the prisms first. Side-boxes never compete.
  • Header chip is a string attribute. Mono-uppercase, cobalt-400, letter-spaced. The component renders it; the user writes one short label.
  • Rows are slotted content. Use any HTML — typically a square icon wrapper + label text. Five rows max; if you need more, switch to compact with a + N more footer.
  • Three widths, no full-width. Side-boxes never stretch. They flank prisms and stay narrower than the prism cluster they support.
  • Cobalt-only ink. No family colours on side-boxes; that visual real estate belongs to the prisms.
