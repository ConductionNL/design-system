Probleemoplossing.

Veelvoorkomende foutmeldingen bij het koppelen van een Open Register-folder, en wat je doet als er iets misgaat.

! The uploaded file exceeds upload_max_filesize

Verhoog upload_max_filesize en post_max_size in php.ini, of stap over op Mode 4 (chunked upload). De default van 2 MB is voor de meeste bijlagen te krap.

! Bestand verschijnt niet in /files-lijst na MOVE

Controleer dat de Destination-header naar Open Registers/<register>/<schema>/<id>/ wijst. Een typo in een van de drie segmenten geeft 201, maar de file landt nergens.

! 403 Forbidden bij PROPFIND op de share

De share staat op "Read only". Zet hem op "Allow editing" in het Files-menu, anders kan de webhook geen bestanden plaatsen.

! Conflict 412: ETag mismatch

Een ander proces heeft de file in de tussentijd geupdate. Refresh de lokale kopie via GET /files/<path>, merge je wijziging, en stuur PUT opnieuw met de nieuwe ETag.

React / MDX usage

Mount in MDX as <Troubleshooting/> + <TroubleshootingItem/>

Structured error-recovery list for academy tutorials and per-app docs. Pairs a literal symptom (usually the error message a user pastes into search) with a one-paragraph body that says what to do. The symptom block lives in a warning-tinted slot on top; the fix fills the rest. Keep symptoms verbatim. Keep fixes under three sentences.

Import & example

import {Troubleshooting, TroubleshootingItem} from '@conduction/docusaurus-preset/components';

<Troubleshooting title="Probleemoplossing" lede="Veelvoorkomende foutmeldingen...">
  <TroubleshootingItem symptom="The uploaded file exceeds upload_max_filesize">
    Verhoog <code>upload_max_filesize</code> in <code>php.ini</code>, of stap over op Mode 4.
  </TroubleshootingItem>
  <TroubleshootingItem symptom="Bestand verschijnt niet in /files-lijst na MOVE">
    Controleer dat de <code>Destination</code>-header naar <code>Open Registers/...</code> wijst.
  </TroubleshootingItem>
</Troubleshooting>

Props

  • title heading text (default Troubleshooting; usually Probleemoplossing in NL)
  • lede optional one-paragraph intro before the list
  • children one or more <TroubleshootingItem/> nodes

TroubleshootingItem takes symptom (string, rendered as monospace) and children (the fix body, MDX inline markup allowed).
